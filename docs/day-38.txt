1. PERFORMANCE PROFILING
----------------------------------------------------------------
Task: Comprehensive Performance Profiling (Memory leaks, frame drops on graphs)

Q: What is Performance Profiling in Flutter?
A: Performance profiling is the process of analyzing your app's performance metrics to identify bottlenecks, jank (frame drops), and memory leaks. In Flutter, we want our apps to run at a smooth 60fps or 120fps.

Q: How do you profile a Flutter app?
A: 
1. Use Profile Mode: NEVER measure performance in Debug mode because it has extra overhead (like assertions and debugging symbols). Use `flutter run --profile`.
2. Flutter DevTools: Use the "Performance" tab in DevTools.
   - UI Thread: Shows if Dart code (build methods, logic) is too slow.
   - Raster Thread: Shows if rendering the UI (shaders, complex clips) is too slow.
3. Identify Jank: If a frame takes longer than 16ms (for 60fps), it causes a dropped frame or "jank".
4. Memory Profiling: Use the "Memory" tab to take a heap snapshot and find memory leaks (e.g., forgotten listeners, timers not canceled, controllers not disposed).

Q: How can we optimize graphs or complex UI in Flutter?
A: 
- Use `const` constructors heavily to prevent unnecessary widget rebuilds.
- Avoid deep widget trees when possible.
- For lists, always use `ListView.builder` (which loads items lazily) instead of `ListView`.
- For graphs (like `fl_chart`), minimize the number of data points rendered at once or use a `RepaintBoundary` around the graph so that when other UI changes, the graph isn't repainted unnecessarily.
- Avoid excessive use of `Opacity`, `ClipPath`, or `BackdropFilter` as they are expensive to render. Use `color: Colors.black.withOpacity(0.5)` instead of an `Opacity` widget when possible.

2. FIREBASE CRASHLYTICS
----------------------------------------------------------------
Task: Implement Firebase Crashlytics for production bug tracking

Q: What is Firebase Crashlytics?
A: It is a real-time crash reporter that helps track, prioritize, and fix stability issues that erode app quality. It tells you exactly what line of code caused a crash and the context around it.

Q: How is it implemented in Flutter?
A: 
1. Add the package: `firebase_crashlytics`.
2. In `main.dart`, catch all Flutter framework errors:
   `FlutterError.onError = FirebaseCrashlytics.instance.recordFlutterFatalError;`
3. Catch all asynchronous errors (Dart errors not caught by the Flutter framework):
   ```dart
   PlatformDispatcher.instance.onError = (error, stack) {
     FirebaseCrashlytics.instance.recordError(error, stack, fatal: true);
     return true;
   };
   ```

Q: Why is Crashlytics important in Production?
A: During development, you see errors in the console. In production, if the app crashes, the user just sees the app close. Without Crashlytics, you are flying blind. Crashlytics groups similar crashes, tells you the OS version and device model, and helps you fix the most critical bugs first.

3. FIREBASE ANALYTICS
----------------------------------------------------------------
Task: Setup Firebase Analytics event logging for all success metrics

Q: What is Firebase Analytics?
A: It provides insight into app usage and user engagement. It automatically tracks core events (like session starts, screen views), and allows logging custom events.

Q: How did we implement it?
A:
We created an `AnalyticsService` containing methods like `logHabitCreated`, `logHabitCompleted`, and `logStreakMilestone`.
```dart
  Future<void> logHabitCompleted(String habitId, String habitTitle) async {
    await _analytics?.logEvent(
      name: 'habit_completed',
      parameters: {
        'habit_id': habitId,
        'habit_title': habitTitle,
      },
    );
  }
```

Q: Why log custom events?
A: Custom events tie directly into Business KPIs (Key Performance Indicators). For StreakSky, knowing how many users complete a habit (`habit_completed`) versus how many create one (`habit_created`) gives us the "Activation Rate". Tracking `streak_milestone` tells us about user retention and engagement. If users aren't reaching milestones, we might need to improve the app's gamification or send better push notifications.

INTERVIEW TIPS & TALKING POINTS
----------------------------------------------------------------
1. Focus on the User: Mention that performance profiling and crashlytics aren't just technical tasks; they directly impact user retention. A slow or crashing app gets uninstalled.
2. The "Safety Net": Explain that Crashlytics is your safety net. "I never deploy to production without a robust crash reporting tool like Crashlytics because..."
3. Data-Driven Decisions: Emphasize that Analytics turns engineering into a business tool. "We implemented Analytics so product decisions can be driven by data (which features are used most) rather than guesses."
