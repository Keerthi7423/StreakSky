INTERVIEW PREPARATION (Q&A)
---------------------------------------------------------
Q: Why did you choose in-memory filtering for the heatmap instead of server-side filtering?
A: Since the user might toggle between different habits frequently, fetching the entire year's completion data once and filtering it in the Riverpod controller provides a much smoother, near-instant user experience compared to making a network request every time a chip is tapped.

Q: How do you handle UI responsiveness when rendering 365+ cells in multiple grids?
A: We used a custom HeatmapGrid widget that utilizes a GridView.count with crossAxisCount: 7. To ensure performance, we use 'const' constructors where possible and only rebuild cells when their specific data changes.

Q: How does the "Multi-year" toggle impact the database load?
A: We fetch data for the specific years defined in the plan. To minimize load, we use indexed columns in Supabase (user_id, completed_date) so that fetching thousands of completion records is extremely fast.
