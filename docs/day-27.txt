3. INTERVIEW PREPARATION (Q&A)
----------------------------------------------------------------

Q: Why use Ollama for on-device AI instead of OpenAI?
A: Privacy and cost. By running Llama 3.2 locally via Ollama, user habit data never leaves their device (or local network), which is crucial for personal privacy apps. It also eliminates API costs for the developer and ensures offline functionality.

Q: How do you handle AI response latency in Flutter?
A: We use a combination of loading states (AsyncValue/isLoading) and optimistic UI updates for the user's message. While the AI is "thinking," we display a pulsing dot animation to keep the interface feeling responsive.

Q: Explain the Riverpod state management for the chat.
A: We use a StateNotifierProvider with an 'AiState' class. This state holds a list of 'ChatMessage' objects. When a user sends a message, we update the state to include the user's message and set 'isLoading: true'. Once the API returns, we append the AI's response and set 'isLoading: false'.

Q: How did you implement the "neon" effect in the UI?
A: We used a combination of high-contrast colors (#B3FF00 on #0D0D0D), BoxShadows with spread and blur to create a glow, and subtle gradients in the nudge cards.
